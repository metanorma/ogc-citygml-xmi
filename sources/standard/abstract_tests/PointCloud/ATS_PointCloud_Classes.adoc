[abstract_test]
====
[%metadata]
标签:: /ats/pointcloud/classes
主体:: <<req_pointcloud_classes,/req/pointcloud/classes>>

[.component,class=测试目的]
--
验证实现规范是否正确实现了概念模型中定义的UML类。
--

[.component,class=测试方法类型]
--
人工检查
--

[.component,class=测试方法]
=====

[.component,class=step]
======
对于点云包中定义或引用的每个UML类：

[.component,class=step]
--
验证实现规范是否包含一个表示与UML类相同概念的数据元素。
--

[.component,class=step]
--
验证该数据元素与其他元素之间的关系是否与UML类定义的关系相同。验证这些关系是否具有与概念模型中记录的相同的源、目标、方向、角色和多重性。
--

[.component,class=step]
--
验证该数据元素是否具有与UML类指定的相同的属性（属性）。验证这些属性是否具有与概念模型中记录的相同的名称、定义、类型和多重性。
--

[.component,class=step]
--
验证该数据元素的属性是否包括概念模型中记录的UML类的所有基类的属性。验证这些属性是否具有与概念模型中记录的相同的名称、定义、类型和多重性。
--

[.component,class=step]
--
验证该数据元素所表示的关联是否包括概念模型中记录的UML类的所有基类的关联。验证这些表示是否具有与概念模型中记录的相同的源、目标、角色和多重性。
--

[.component,class=step]
--
验证实现规范是否强制执行概念模型对UML类施加的所有约束。
--
