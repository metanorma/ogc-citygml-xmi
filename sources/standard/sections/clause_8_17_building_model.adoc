建筑模块用于呈现建筑物的主题和空间方面。建筑物是独立、自承重的结构，通常有屋顶和墙壁，可供人类进入，且一般设计为永久性地位于一个地方。
建筑物旨在供人类居住（例如工作或娱乐场所）、居住以及 / 或者为人类、动物或物品提供遮蔽。在 UML 模型中，建筑物由顶级要素类型 _Building_ 表示，它是建筑模块的主要类。建筑物可以在物理上或功能上细分为建筑部件，并在逻辑上细分为楼层和建筑单元（例如公寓）。此外，建筑物可以分解为结构元素，如墙壁、楼板、楼梯和梁。


建筑物的内部由房间来表示。这使得建筑物具有虚拟可访问性，例如用于博物馆的游客信息（“基于位置的服务”）、检查住宿标准或展示建筑物的日光照明情况。

建筑物可以包含设施和家具。设施是建筑物的永久性部分，对建筑物的外观或内部有显著影响且无法移动。例如阳台、烟囱、老虎窗或楼梯。相比之下，家具代表建筑物内的可移动物体，如桌子和椅子。

建筑物可以由不同类型的表面界定。通过这种方式，建筑物的外立面可以在语义上细分为墙面、屋顶面、地面、外部楼面和外部天花板面，而房间的可见表面可以细分为内部墙面、楼面和天花板面。此外，建筑物的开口，即窗户和门，可以连同其相应的表面一起表示。

建筑模块的 UML 图如 <<building-uml>>. 建筑模块继承自 Construction 模块的概念 (cf. lutaml_table::[package="Construction"]). Construction 模块定义了所有类型建筑共有的对象，例如不同的表面类型和开口。关于需求类 Building 的详细讨论可以在 link:http://docs.opengeospatial.org/DRAFTS/20-066.html#ug-model-building-section[CityGML 3.0 Users Guide]中找到。
