桥梁模块提供了对桥梁主题和空间方面的表示。桥梁是跨越中间自然或人造元素的可移动或不可移动结构。通过这种方式，桥梁允许行人、动物、车辆和服务在障碍物上方或高于地面的两点之间通过。在 UML 模型中，桥梁由顶级要素类型 _Bridge_ 表示，它是桥梁模块的主要类。
桥梁可以在物理上或功能上细分为桥梁部分。此外，桥梁可以分解为结构元素，如桥墩、锚碇、缆索、板和梁。


桥梁内部的自由空间由房间表示，这使得可以对桥梁进行虚拟访问。桥梁可以包含设施和家具。设施是桥梁的永久部分，对桥梁的外部或内部外观有很大影响且无法移动。例如楼梯、信号灯、栏杆和灯具。相比之下，家具代表桥梁的可移动物体，如标志、艺术品和长椅。桥梁可以由不同类型的表面界定。通过这种方式，桥梁的外部结构在语义上可以细分为墙面、屋顶面、地面、外部楼面和外部天花板面，而房间的可见表面可以细分为内部墙面、楼面和天花板面。此外，桥梁的开口，即窗户和门，可以连同其相应的表面一起表示。

桥梁模块的 UML 图如 <<bridge-uml>>所示。桥梁模块继承自建筑模块的概念 (cf. lutaml_table::[package="Construction"]). 建筑模块定义了所有类型建筑共有的对象，如不同的表面类型和开口。关于需求类桥梁的详细讨论可以在 link:http://docs.opengeospatial.org/DRAFTS/20-066.html#ug-model-bridge-section[CityGML 3.0 Users Guide]中找到。
