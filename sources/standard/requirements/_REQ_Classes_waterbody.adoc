以下需求定义了将 CityGML 水体模块作为实施规范的实施规则。

[[req_waterbody_classes]]
[requirement,type="general",label="/req/waterbody/classes"]
====
对于在水体包中定义或引用的每个 UML 类：

[.component,class=part]
--
实施规范应包含一个元素，该元素表示与 UML 类定义相同的概念。
--

[.component,class=part]
--
实施规范应表示关联，其源、目标、方向、角色和多重性与 UML 类的相同。
--

[.component,class=part]
--
实施规范应表示 UML 类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的关联，包括源、目标、方向、角色和多重性。
--

[.component,class=part]
--
实施规范应指定实现如何遵守概念模型对 UML 类施加的所有约束。
--
====
