以下需求定义了将CityGML点云模块作为实施规范进行实施的规则。

[[req_pointcloud_classes]]
[requirement,type="general",label="/req/pointcloud/classes"]
====
对于在点云包中定义或引用的每个UML类：

[.component,class=part]
--
实施规范应包含一个元素，其表示的概念与为UML类定义的概念相同。
--

[.component,class=part]
--
实施规范应以与UML类相同的源、目标、方向、角色和多重性来表示关联。
--

[.component,class=part]
--
实施规范应表示UML类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示UML类所有基类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示UML类所有基类的关联，包括源、目标、方向、角色和多重性。
--

[.component,class=part]
--
实施规范应指定实现如何遵守概念模型对UML类施加的所有约束。
--
====
