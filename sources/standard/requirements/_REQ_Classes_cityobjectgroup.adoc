以下需求定义了将 CityGML 城市对象组模块作为实施规范进行实施的规则。

[[req_cityobjectgroup_classes]]
[requirement,type="general",label="/req/cityobjectgroup/classes"]
====
对于在城市对象组包中定义或引用的每个 UML 类：

[.component,class=part]
--
实施规范应包含一个元素，其表示的概念与为 UML 类定义的概念相同。
--

[.component,class=part]
--
实施规范应以与 UML 类相同的源、目标、方向、角色和多重性来表示关联。
--

[.component,class=part]
--
实施规范应表示 UML 类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的关联，包括源、目标、方向、角色和多重性。
--

[.component,class=part]
--
实施规范应指定实现如何遵守概念模型对 UML 类施加的所有约束。
--
====
