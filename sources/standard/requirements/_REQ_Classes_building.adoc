以下需求定义了将 CityGML 建筑模块作为实施规范进行实施的规则。

[[req_building_classes]]
[requirement,type="general",label="/req/building/classes"]
====
对于建筑包中定义或引用的每个 UML 类：

[.component,class=part]
--
实施规范应包含一个元素，其表示与 UML 类所定义的相同概念。
--

[.component,class=part]
--
实施规范应表示关联，其源、目标、方向、角色和多重性与 UML 类的相同。
--

[.component,class=part]
--
实施规范应表示 UML 类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示UML 类所有基类的关联，包括源、目标、方向、角色和多重性。
--

[.component,class=part]
--
实施规范应指定一个实施如何遵守概念模型对 UML 类施加的所有约束。
--
====
