以下需求定义了将 CityGML 城市家具模块作为实施规范进行实施的规则。

[[req_cityfurniture_classes]]
[requirement,type="general",label="/req/cityfurniture/classes"]
====
对于在城市家具包中定义或引用的每个 UML 类：

[.component,class=part]
--
实施规范应包含一个元素，该元素代表与为 UML 类定义的概念相同的概念。
--

[.component,class=part]
--
实施规范应表示关联，其源、目标、方向、角色和多重性与 UML 类的相同。
--

[.component,class=part]
--
实施规范应表示 UML 类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的属性，包括名称、定义、类型和多重性。
--

[.component,class=part]
--
实施规范应表示 UML 类所有基类的关联，包括源、目标、方向、角色和多重性。
--

[.component,class=part]
--
实施规范应指定实现 F 如何遵守概念模型对 UML 类施加的所有约束。
--
====
